# MemWeave

**Local-first, memorable, reasoning-capable memory infrastructure for AI agents.**

## OVERVIEW

TypeScript monorepo with two apps: a **Fastify + SQLite memory server** (`src/`) and a **React/Vite admin UI** (`web/`). The server exposes REST + MCP + an OpenCode plugin; the UI is a 7-page "Calm Memory Atlas" for browsing memories, viewing injection previews, and inspecting consolidation runs.

## CONVENTIONS

- **TypeScript strict ESM** (`"type": "module"`, `module: NodeNext`). All imports use `.js` suffix even for `.ts` files.
- **Zod-first types**: every enum (`MemoryType`, `MemoryTier`, `EdgeType`, etc.) lives in `src/core/types.ts` as a Zod schema; types are inferred.
- **One file per route/tool/command**: each REST route, MCP tool, and CLI subcommand is its own file. `index.ts` files re-export.
- **Idempotent schema**: `SCHEMA_SQL` uses `IF NOT EXISTS`. To migrate, add a `migrations/` step (currently a `migrate` command runs the schema verbatim).
- **No external LLM required**: `embedding.dimensions: 0` disables the vector layer; `providers/llm/noop.ts` makes consolidation a pure rule-based pass.
- **Multi-tenant by default**: every repository method takes `tenantId`; `auth.ts` middleware validates API key against `tenants.api_key_hash`.

## ANTI-PATTERNS (THIS PROJECT)

- **NEVER** add `as any` / `@ts-ignore` — strict TS is enforced in CI-equivalent (`npm run typecheck`).
- **NEVER** edit `dist/` by hand — it's the tsc + Vite output. Run `npm run build` instead.
- **NEVER** call the SQLite DB outside `src/db/repositories/*`. The rest of the codebase talks to repositories only.
- **NEVER** import from `dist/` — it's build output, not a dependency.
- **NEVER** run the OpenCode plugin (`src/plugin/`) in the same process as the server — the plugin uses `@opencode-ai/plugin` which expects to run inside OpenCode's hook context.
- **DO NOT** add a new top-level `src/<feature>/` directory without updating this AGENTS.md.
- **DO NOT** introduce a different web framework — Vite + React 18 is the only frontend stack.

## UNIQUE STYLES

- **"Calm Memory Atlas"** is the project's design language: warm-paper light theme (`#F7F4EE`) + sage-green accent (`#3B7C6E`). See `web/src/theme/tokens.css`.
- **Tier-based memory lifecycle**: `short` → `medium` → `long` via consolidation. Decay modeled in `src/core/decay.ts` (tau + reinforcement_score).
- **Edge type vocabulary** (`src/core/types.ts`): `causes`, `enables`, `contradicts`, `supersedes`, `references`, `related_to`, `before`, `after`, `duplicates`, `refines` — design §10.6 colors are defined in `tokens.css`.
- **RRF fusion** (not learned reranking): `src/retrieval/fusion.ts` uses Reciprocal Rank Fusion with configurable K (default 60).
- **Plugin is fail-silent**: `src/plugin/injector.ts` catches all errors from the server so a MemWeave outage never breaks the agent.

## COMMANDS

```bash
# Setup
npm install
npm run cli -- init                # creates memweave.config.jsonc + data dir

# Dev
npm run dev                        # builds web + starts server on :3131
npm run web:dev                    # Vite HMR on :5173 (proxies /api → :3131)

# Build
npm run build                      # web → dist/web/ + tsc → dist/
npm run typecheck                  # tsc --noEmit (server)

# Test
npm test                           # server vitest (requires server autostart)
cd web && npm test                 # frontend vitest (happy-dom)

# CLI
npm run cli -- start               # foreground
npm run cli -- stop                # via PID file
npm run cli -- doctor              # health self-check
npm run mcp                        # stdio MCP server
```

## NOTES

- **Default port**: 3131 (configurable in `memweave.config.jsonc → server.port`).
- **Web UI 503?** Means the server can't find `dist/web/` — run `npm run web:build`.
- **Vector layer optional**: `embedding.dimensions: 0` skips sqlite-vec entirely; falls back to BM25 + graph + causal.
- **No git history**: this is a fresh checkout; commit/branch conventions not yet established.
- **Docs**: design specs live in `docs/superpowers/specs/`. The README is the canonical feature overview.
